```
LAUDO_SUS_PE       → 🟢 Verde escuro
LAUDO_SUS_GERAL    → 🟢 Verde claro
AUX_DOENCA         → 🟣 Roxo
AUX_ACIDENTE       → 🟠 Laranja
PERICIA_NEGADA     → 🔴 Vermelho
```

**Onde aplicar:**
- Badge colorido no card de cada lead no dashboard
- Borda lateral colorida no card
- Legenda de cores no canto do dashboard
- Gráfico de pizza usando essas cores

Criar um arquivo `lib/api-client-react/src/utils/campaignColors.ts` exportando o mapa de cores e funções helper.

### 2️⃣ BADGE "NOVO" PISCANDO

Leads que chegaram nas últimas **30 minutos** ganham um badge animado "NOVO" com efeito pulse/glow.

- Cor: vermelho ou amarelo neon
- Animação CSS sutil (pulse a cada 2s)
- Visível no card do lead no dashboard
- Some automaticamente após 30 minutos
- Tooltip: "Lead recebido há X minutos"

Implementar com CSS keyframes ou framer-motion (se já tiver instalado).

---

### 3️⃣ BUSCA RÁPIDA (Ctrl+K / Cmd+K)

Atalho global em todo o sistema:

- Pressionar **Ctrl+K** (Windows/Linux) ou **Cmd+K** (Mac) abre um modal de busca
- Campo de input com foco automático
- Busca em tempo real conforme digita (debounce 300ms)
- Busca por: nome do contato, telefone, primeira mensagem
- Resultado clicável que leva pra ficha do lead
- Botão "Esc" pra fechar

**Endpoint novo:**
```
GET /api/conversations/search?q=termo
```
Retorna leads que batem com o termo (no nome OU no telefone OU na primeira mensagem).

---

### 4️⃣ EXPORTAR PRA CSV

Botão "Exportar CSV" no dashboard que:

- Respeita os filtros aplicados (período, campanha, status, etc.)
- Gera arquivo CSV com colunas: ID, Nome, Telefone, Número Recebeu, Campanha, Status, Atendente, Tags, Primeira Mensagem, Data Criação, Data Última Mensagem
- Nome do arquivo: `leads_eduardo_rodrigues_AAAA-MM-DD.csv`
- Download direto pelo navegador

**Endpoint novo:**
```
GET /api/conversations/export?filters...
```
Retorna content-type `text/csv`.

### 5️⃣ ANOTAÇÃO RÁPIDA EM CADA LEAD

Campo de texto livre na ficha do lead pra observações.

**Schema novo (se necessário):**
Adicionar campo `notes` (text, nullable) na tabela `conversations`.

**UI:**
- Textarea expansível na ficha do lead
- Auto-save a cada 2 segundos sem mexer (debounce)
- Indicador visual "Salvando..." / "Salvo ✓"
- Suporta múltiplas linhas
- Conta de caracteres (limite 2000)

**Endpoint:**
```
PATCH /api/conversations/:id
Body: { notes: "texto livre" }
```

### 6️⃣ TIMELINE DE STATUS

Histórico visual de mudanças de status de cada lead.

**Schema novo:**
Criar tabela `status_history`:
```typescript
export const statusHistoryTable = pgTable("status_history", {
  id: serial("id").primaryKey(),
  conversationId: integer("conversation_id").notNull().references(() => conversationsTable.id),
  fromStatus: text("from_status"), // null se foi a primeira atribuição
  toStatus: text("to_status").notNull(),
  changedBy: text("changed_by"), // "system" / "manual" / "chatguru_sync"
  notes: text("notes"), // opcional
  createdAt: timestamp("created_at").defaultNow().notNull(),
});
```

**Lógica:**
- Toda vez que o status muda (manual ou via sync ChatGuru), registra evento
- Toda vez que adiciona ou remove tag de STATUS, registra

**UI:**
- Componente "Timeline" na ficha do lead
- Linha vertical com pontos (cada mudança = um ponto)
- Mostra: "LEAD NOVO → LEAD QUALIFICADO" há 3h
- Cores diferentes por status
- Tempo entre cada mudança visível

### 8️⃣ ALERTA DE LEAD ESFRIANDO

Sistema identifica leads "parados" e alerta.

**Regras:**
- LEAD QUALIFICADO sem mudança de status há mais de **24h** = "esfriando"
- FOLLOW UP sem nova mensagem há mais de **48h** = "esfriando"
- LEAD NOVO sem nenhuma resposta da equipe há mais de **2h** = "URGENTE"

**Onde aparece:**
- Card de "Alertas" no topo do dashboard com contador
- Badge especial nos leads esfriando (ícone de fogo apagando 🥶 ou termômetro)
- Página dedicada `/alerts` lista todos os leads em alerta
- Filtro rápido no dashboard: "Mostrar só alertas"

**Cron job:**
- Roda a cada 30 minutos
- Marca leads como "esfriando" se atender critérios
- Limpa flag se houver atividade nova

### 9️⃣ RESUMO DIÁRIO AUTOMÁTICO

Toda noite às **20:00**, sistema gera um resumo do dia.

**Conteúdo do resumo:**
```
📊 RESUMO DO DIA — 26/04/2026

🆕 LEADS NOVOS HOJE: 12
├─ Tráfego Comercial: 9
└─ Base: 3

📞 POR CAMPANHA:
├─ AUX DOENÇA: 4
├─ AUX ACIDENTE: 3
├─ LAUDO SUS PE: 2
├─ PERÍCIA NEGADA: 2
└─ LAUDO SUS GERAL: 1

✅ MOVIMENTAÇÃO:
├─ Qualificados: 6
├─ Em follow-up: 4
├─ Contratos assinados: 2
└─ Descartados: 3

🏆 DESTAQUE: Thiago fechou 2 contratos hoje!

⚠️ ALERTAS:
├─ 3 leads esfriando
└─ 1 lead urgente sem resposta
```

**Como entregar:**
- Salvar o resumo na tabela `daily_summaries` (data + conteúdo HTML/JSON)
- Página `/summaries` lista todos os resumos diários
- Card no topo do dashboard com "Resumo de ontem"
- **OPCIONAL avançado:** enviar via webhook pra um número de WhatsApp pessoal do Eduardo (futuramente)

**Schema:**
```typescript
export const dailySummariesTable = pgTable("daily_summaries", {
  id: serial("id").primaryKey(),
  date: text("date").notNull().unique(), // "2026-04-26"
  data: jsonb("data").notNull(), // estrutura completa do resumo
  generatedAt: timestamp("generated_at").defaultNow().notNull(),
});
```

**Cron:**
- Roda todo dia às 20:00 (horário de Brasília)
- Gera resumo do dia que está acabando
- Salva no banco
