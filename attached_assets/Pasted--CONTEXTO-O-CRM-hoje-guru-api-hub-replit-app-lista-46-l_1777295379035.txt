# CONTEXTO

O CRM hoje (guru-api-hub.replit.app) lista 46 leads urgentes na aba **Alertas**, mas **não mostra a mensagem que o lead enviou**. Isso obriga o Eduardo a clicar 46 vezes pra abrir o ChatGuru e descobrir do que se trata cada lead — inviável.

A integração com ChatGuru já existe (endpoint `https://s22.chatguru.app/api/v1`, chave configurada). Vamos usar ela pra duas coisas:

1. **Mostrar preview** das últimas 2-3 mensagens da conversa direto nos cards de lead
2. **Enviar mensagens prontas** com 1 clique, sem sair do CRM

## PARTE 1 — PREVIEW DE MENSAGENS

### 1.1 — Backend: criar endpoint `/api/conversations/:chatNumber/messages`

Criar handler que:

1. Recebe `chatNumber` na URL (ex: `554488562824`)
2. Chama API do ChatGuru pra buscar o histórico de mensagens dessa conversa
3. Retorna as **últimas 5 mensagens** (vamos exibir 2-3 mas guardar 5 pra hover)
4. Cada mensagem com: `texto`, `direção` (lead/equipe), `autor` (nome de quem mandou se for da equipe), `timestamp`

**Resposta esperada:**

```json
{
  "chatNumber": "554488562824",
  "contactName": "Daniel Guedes",
  "messages": [
    {
      "text": "oi, queria saber sobre o laudo do meu pai. ele tem alzheimer e a perícia foi negada",
      "direction": "incoming",
      "author": "Daniel Guedes",
      "timestamp": "2026-04-26T14:32:00Z"
    },
    {
      "text": "Oi Daniel! Que situação difícil 🙏 Me conta...",
      "direction": "outgoing",
      "author": "Thiago Tavares",
      "timestamp": "2026-04-26T14:35:00Z"
    },
    {
      "text": "ele tem 67 anos, mora comigo, eu cuido dele",
      "direction": "incoming",
      "author": "Daniel Guedes",
      "timestamp": "2026-04-26T14:38:00Z"
    }
  ]
}
```

**Cache:** 2 minutos (mensagens novas chegam rápido, não vale cachear muito).

> **Importante:** se a API do ChatGuru tiver endpoint específico (procurar nos docs), use. Se não, busca via `webhook_events` no banco local — você já tem armazenado.

## PARTE 2 — BOTÕES DE MENSAGEM PRONTA

### 2.1 — Backend: criar endpoint `/api/conversations/:chatNumber/send`

Cria handler que:

1. Recebe `chatNumber` na URL
2. Body: `{ "templateId": "A" | "B" | "C" | "D", "customMessage": "..." }`
3. Busca dados do lead (nome, campanha) no banco local
4. Renderiza o template substituindo variáveis (`{nome}`, `{campanha}`)
5. Envia via API do ChatGuru
6. Loga o envio numa tabela `sent_messages` (pra auditoria)
7. Retorna sucesso/erro

**Tabela `sent_messages`:**
```typescript
export const sentMessagesTable = pgTable("sent_messages", {
  id: serial("id").primaryKey(),
  chatNumber: text("chat_number").notNull(),
  templateId: text("template_id").notNull(),
  messageText: text("message_text").notNull(),
  sentBy: text("sent_by").notNull(), // sempre "Eduardo" por enquanto
  sentAt: timestamp("sent_at").defaultNow().notNull(),
  chatguruResponse: jsonb("chatguru_response"),
  status: text("status").notNull(), // "success" | "error"
});
```

### 2.2 — Definir os 4 templates

Criar arquivo `server/message-templates.ts`:

```typescript
